# Notification protocol

시간대는 `Asia/Seoul`이다. cron 세션은 독립 실행으로 간주하고 대화 문맥을 신뢰하지 않는다. 매번 `AGENTS.md`, 현재 JSON, `brief.md`, Calendar index를 읽는다.

## 공통 사전 확인

cron은 메시지를 보내기 전에 아래를 먼저 확인한다:

1. `git log --since="today 00:00:00+09:00" --oneline`을 실행한다.
2. 오늘 커밋이 있으면 사용자가 이미 작업하고 있다는 신호다. 아래 규칙을 적용한다:
   - **절대 "뭐할거야", "계획 정할래?" 같은 질문을 하지 않는다.**
   - 대신 오늘 한 작업을 짧게 요약하고 "하고 있는 거 잘 되고 있어?" 정도로 물어본다.
   - 생활일이 없으면 "생활일 시작 안 했네. 지금 하고 있는 작업 기록할래?"로 제안한다.
   - "오늘 뭐 먹었어?" 같은 일상 질문은 여전히 해도 된다.

알릴 내용이 없으면 `[SILENT]`로 끝낸다.

## 20:00 — status check

### 활성 생활일이 있을 때

- 오늘 태스크가 정해졌으면 진행 상태를 부담 없이 확인한다.
- 오늘 태스크가 없고 `workday_capacity > 0`이면:
  - 오늘 git 커밋이 있으면 → "지금 하는 작업 계획에 넣을까?" 정도로 제안. "뭐할거야" 금지.
  - 오늘 git 커밋이 없으면 → 계획을 정할지 묻는다.
- 이미 작업 세션이 열려 있으면 중복 체크인을 유도하지 않는다.
- 미완료 pending이 있으면 리마인드한다.
  - "밀린 일 리마인드야: ..." 정도로 짧게.
- 오늘 meals가 하나도 기록되지 않았으면 가볍게 물어본다.
  - "오늘 뭐 먹었어?" — 답변을 강요하지 않고 넘긴다.

### 활성 생활일이 없을 때

- 생활일 시작 신호를 놓쳤을 가능성을 부드럽게 확인한다.
- 사용자가 잘못했다고 단정하지 않는다.
- 새 생활일을 자동 생성하지 않는다.

## 22:50 — work start check

다음 조건을 모두 만족할 때만 보낸다.

- 활성 생활일 있음
- `workday_capacity > 0`
- 계획된 태스크 있음
- 열린 작업 세션 없음

메시지에는 태스크명과 첫 행동만 짧게 보여준다. 이미 작업 중이면 `[SILENT]`.

## 03:00 — checkout check

다음 조건을 모두 만족할 때만 보낸다.

- 활성 생활일 있음
- 해당 생활일에 체크인 기록이 있음
- 열린 작업 세션이 있음

완료한 것, 막힌 점, 태스크 상태, 다음 첫 행동을 확인한다. 새 태스크를 권하지 않는다. 열린 세션이 없으면 `[SILENT]`.

### 03:00에 함께 확인할 것 (세션이 열려 있을 때만)

- 오늘 mood가 기록되지 않았으면 "오늘 기분 어땠어?"
- 진행 중인 외출이 안 돌아온 상태면 확인
- medical 업데이트가 필요하면 물어본다

## cron 안전 규칙

- cron은 상태를 임의 변경하지 않는다.
- 사용자가 응답한 사실을 기록할 때만 `neoctl`을 실행한다.
- Calendar 쓰기는 cron에서 실행하지 않는다.
- 위험 명령이나 `--approve`가 필요한 명령은 cron에서 실행하지 않는다.
- cron은 todolist/pending add까지만 하고, done/toggle은 하지 않는다.
