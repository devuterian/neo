# Repository inbox

This page is for untriaged repository-development ideas, defects, and documentation gaps. It is not the user's daily todo list and must not contain life-day contents, private events, message-log excerpts, credentials, or operational secrets.

## How to use this inbox

- Capture repository-development ideas that are not yet accepted plans.
- Move accepted work to [`PLANS.md`](PLANS.md) or an issue/PR.
- Move durable decisions to [`docs/adr/`](../docs/adr/) when an architecture decision is accepted.
- Remove items that are obsolete or out of scope.
